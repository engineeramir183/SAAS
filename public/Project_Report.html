Executive Summary
This document serves as the formal technical project report for the development, implementation, and deployment of the ACS School Management ERP System. The platform was designed to transition the institution from manual spreadsheet-based tracking to a fully digitized, centralized, and secure cloud environment handling Operations, Academics, Financial Accounting, and Public Communications.

1. Technology Stack & Architecture

The ACS ERP system is built upon a modern, highly scalable JavaScript web architecture designed for maximal performance, offline capability, and cross-device responsiveness.

Frontend Architecture

The client side is orchestrated using React.js via the ultra-fast Vite bundler. To ensure snap-fast navigation between the 10 core administrative modules, components are Lazily Loaded via React Suspense.

Backend & Database Infrastructure

The system utilizes Supabase, an open-source Firebase alternative, acting as a powerful PostgreSQL database and authentication provider. Data structures that require extreme flexibility—such as tracking a student's partial fee payments, specific exam results across dozens of terms, and daily attendance—are stored efficiently using PostgreSQL JSONB column arrays.

2. Core System Modules

Admissions & Roster Management

Handles complete intake of student data including Name, B-Form, address, and profile photography. Features bi-directional Excel (XLSX) import/export functionality with auto-merging safety checks to prevent duplicates using exact Serial Numbers and CNIC mapping.

Automated ID Generator

Employs dynamic HTML/CSS rendering inside the browser's Document Object Model to compile precise 54mm x 86mm physical ID cards for any selected class batch simultaneously, bypassing the need for third-party PDF generators.

Advanced Fee & Billing Engine

Breaks down standard static charges into granular categories (Tuition, Annual, Transport). Incorporates manual fields for Concessions/Discounts and Late Fines. Actively tracks Net Payable vs. Paid Amount, persisting a "Balance Due" state object mapped to the specific month.

Defaulter Analysis & Reporting

Executes an algorithmic scan across the `feeHistory` array of the entire student database. Aggregates any object possessing a 'partial' or 'unpaid' status flag, ultimately returning a precise, printable Data Table of strictly defaulting entities coupled with their parent contact protocols via 1-Click WhatsApp API integration.

Expense Ledger (Net Profit Calculation)

Combines total historical revenue collection against an internal `EXPENSES` metadata array. Generates a live Profit & Loss (P&L) calculation using a React map-reduce aggregator displaying total Gross Income vs. Total Capital Expenditure.

Academic Gradebook Dashboard

Utilizes dynamic table headers generated from a globally stored `SUBJECTS` mapping. It isolates term boundaries to calculate overall percentages using configured max-weights per subject. Culminates into a fully formatted, printable Parent-Teacher Meeting Report Card execution.

4. Security & Performance Implementations

  • Role-Based Access Control (RBAC): The frontend strictly checks the `adminCredentials` state before mounting the React Router for the `/admin` path, redirecting unauthorized traffic cleanly to the login endpoint.
  • Client-Side Processing: Bulk operations like Excel parsing (`xlsx`), ID card generation, and statistical aggregation are handled by the client's internal V8 browser engine (JavaScript). This mathematically reduces server overhead to almost zero during demanding computational tasks, limiting database calls purely to basic UPSERT logic.
  • Print CSS Engineering: Media query protocols ensure the browser's Document View strictly isolates target elements (e.g. `print-color-adjust: exact`) to produce native PDF generations cleanly without expensive third-party Server-Side-Rendering dependencies.
